# SUMMARY
In this demo we have an "old" ski lift API that we want to migrate to a new API rebuilt on a new stack. The demo should showcase this by running an "old" and a "new" API in parallel. The test project uses Verify to get snapshots from both APIs and verify that they are the same.

The respones should include some response unique data like guids and timestamps to showcase how you can scrub that data with Verify.

## Covered concepts are
* Using Verify snapshot testing to verify that responses from old and new API are the same.
